            <div class="l-grid-10 m-grid-9 s-grid12 content-column"><h1 class="">What is Static Content?</h1></div>
        </div>
        <div class="no-overflow row">
            <div class="l-grid-2 m-grid-3 s-grid-12">
                <div class="article-info">
                    <p class="author-name">Robert Gibb</p>
                    <p class="timestamp">April 27, 2016</p>
                </div>
                <div class="edited-date">
                    <p class="no-bottom-margin">Updated</p>
                    <p >May 13, 2016</p>
                </div>
            </div>
            <div class="l-grid-10 m-grid-9 s-grid-12 content-column post-content">
                <div class="vg-header">
<p>Definition</p>
</div>
<p>Static content is any content that can be delivered to an end user without having to be generated, modified, or processed. The server delivers the same file to each user, making static content one of the simplest and most efficient content types to transmit over the Internet.</p>
<p><img src="https://www.maxcdn.com/one/media/static-content.png" alt="Diagram of how static content works"></p>
<div class="vg-header">
<h2>Overview</h2>
</div>
<p>Most modern websites are always changing. Websites frequently adapt their layout, content, and available features based on who accesses it. In many cases, a website will deliver entirely different content to one user than another user, even if their requests are less than a second apart. This content is known as dynamic content, since it can change on a per-request basis.</p>
<p>Static content, on the other hand, rarely changes. It doesn&#8217;t depend on user input or preferences. While static content is less interactive than dynamic content, it&#8217;s generally faster to cache, process, and deliver.</p>
<div class="vg-header">
<h2>How Static Content Works</h2>
</div>
<p>Fetching a static asset from a server is one of the basic functions of the web. For example, typing the following URL in a web browser (<em>http://www.example.com/index.html</em>) fetches the file <em>index.html</em> from the server hosting <em>example.com</em>. </p>
<p>There are three steps to requesting static content from a server:</p>
<ol>
<li>A user sends a request for a file to the web server.</li>
<li>The web server retrieves the file from disk.</li>
<li>The web server sends the file to the user.</li>
</ol>
<p>Any kind of file can be served as static content as long as it does not change in response to a user&#8217;s actions or inputs. This includes images, JavaScript files, CSS files, videos, Flash files, even web pages.</p>
<div class="vg-header">
<h2>Benefits of Static Content</h2>
</div>
<p>This section isn&#8217;t meant to promote static over dynamic content. Almost every website uses a mix of content types to provide a balance between speed and interactivity. However, there are some clear benefits to hosting static content:</p>
<ul>
<li><strong>Static content does not change.</strong> Once a static file is uploaded to a server, it does not change until you replace it with another file. In the meantime, users who return to your website will see exactly the same content.</li>
<li><strong>Static content is easier to cache.</strong> Although there are tricks to caching dynamic content, it often can&#8217;t be cached effectively because it&#8217;s hard to predict when it&#8217;s needed. Since static content is the same for all users, it can be cached very easily.</li>
<li><strong>Static content is less power-hungry.</strong> Dynamic websites contain layers of application logic that run before the user receives a response. Static websites only need to pull files from the disk. Additionally, techniques such as compression only need to be applied once to static content, making it very resource-efficient.</li>
</ul>
<div class="vg-header">
<h2>Conclusion</h2>
</div>
<p>While static websites drove the initial growth of the web, most of today&#8217;s websites are dynamic, user-driven applications. Although the days of the early web are over, some web services are working to merge the performance gains of static websites with the usability of modern web platforms. </p>
<p><a href="https://www.staticgen.com/" target="_blank">StaticGen</a> provides a leaderboard of website generators that build static websites from dynamic content. Meanwhile, platforms like WordPress support static content generation through plugins like the <a href="https://wordpress.org/plugins/wp-super-cache/" target="_blank">WP Super Cache plugin</a>. It&#8217;s unlikely we&#8217;ll revert back to static web pages, but we can take advantage of the speed and simplicity of static content to boost the modern web.</p>
